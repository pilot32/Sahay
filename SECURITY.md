# 🔒 Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest (`main`) | ✅ Yes |
| Older releases | ❌ No |

---

## 🚨 Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub Issues.**

If you discover a security vulnerability in Sahay, please report it responsibly by emailing:

📧 **security@sahay.org** *(replace with your actual email)*

### What to include in your report:
- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact (what an attacker could do)
- Any suggested fix (optional but appreciated)

---

## ⏱️ Response Timeline

| Step | Timeframe |
|------|-----------|
| Acknowledgement of report | Within 48 hours |
| Initial assessment | Within 5 business days |
| Fix & patch release | Within 30 days (critical: ASAP) |
| Public disclosure | After fix is released |

---

## 🙏 Responsible Disclosure

We ask that you:
- Give us reasonable time to fix the issue before public disclosure
- Avoid accessing, modifying, or deleting data that isn't yours
- Do not exploit the vulnerability beyond what is needed to demonstrate it

We commit to:
- Keeping you informed throughout the process
- Crediting you in our security acknowledgements (if you wish)
- Not taking legal action against good-faith security researchers

---

Thank you for helping keep Sahay and its users safe. 💚
